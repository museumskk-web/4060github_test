좋아, 바로 수업에 쓸 수 있는 **1차시 GitHub 교육자료 (Markdown)** 만들어줄게. 그대로 복붙해서 `.md` 파일로 쓰면 돼.

````markdown
# GitHub 사용법 1차시 교육자료
## (기초 개념 + 저장소 생성 + 기본 사용)

---

## 🎯 학습 목표
- Git과 GitHub의 차이를 이해한다
- GitHub 계정을 생성하고 저장소를 만든다
- 파일 업로드 및 기본 관리 방법을 익힌다

---

## 1. Git과 GitHub 이해하기

### 📌 Git이란?
- 버전 관리 시스템 (Version Control System)
- 파일의 변경 이력을 저장하고 관리
- 이전 상태로 되돌리기 가능

### 📌 GitHub란?
- Git 저장소를 온라인에서 관리하는 플랫폼
- 협업을 위한 기능 제공 (공동 작업, 코드 공유 등)

### 💡 쉽게 이해하기
- Git = 작업 기록을 저장하는 도구
- GitHub = 그 기록을 올려두는 클라우드 공간

---

## 2. GitHub 계정 생성

### ✔️ 가입 절차
1. https://github.com 접속
2. "Sign up" 클릭
3. 이메일, 비밀번호, 사용자 이름 입력
4. 인증 절차 진행

---

## 3. 저장소(Repository) 만들기

### 📌 저장소란?
- 프로젝트 파일을 저장하는 공간

### ✔️ 생성 방법
1. 로그인 후 오른쪽 상단 "+" 클릭
2. "New repository" 선택
3. 아래 내용 입력
   - Repository name: 프로젝트 이름
   - Public / Private 선택
   - README 체크 (권장)
4. "Create repository" 클릭

---

## 4. 파일 업로드하기

### ✔️ 방법 1: 웹에서 직접 업로드
1. 저장소 접속
2. "Add file" → "Upload files"
3. 파일 드래그 또는 선택
4. "Commit changes" 클릭

---

## 5. Commit 개념 이해하기

### 📌 Commit이란?
- 변경 사항을 기록하는 작업

### 💡 예시
- "README 파일 수정"
- "이미지 추가"

### ✔️ 좋은 Commit 메시지 작성법
- 간단하고 명확하게 작성
- 예:
  - "Add login feature"
  - "Fix typo in README"

---

## 6. README 파일 이해하기

### 📌 README란?
- 프로젝트 설명서 역할
- Markdown(.md) 형식 사용

### ✔️ 작성 예시
```markdown
# 프로젝트 이름

## 소개
이 프로젝트는 GitHub 연습용입니다.

## 기능
- 기능 1
- 기능 2
````

---

## 7. Markdown 기본 문법

### ✔️ 제목

```
# 제목1
## 제목2
### 제목3
```

### ✔️ 목록

```
- 항목1
- 항목2
```

### ✔️ 강조

```
**굵게**
*기울임*
```
